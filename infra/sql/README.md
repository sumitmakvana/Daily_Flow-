# SQL Files

- `00-auth-stub.sql` — committed. Creates `auth.uid()`/`auth.role()`/etc and
  the PostgREST roles. Apply BEFORE GoTrue boots so GoTrue's own migrations
  land in the same schema cleanly.
- `01-schema.sql`, `02-data.sql`, `03-policies.sql` — **generated by
  `scripts/export-from-supabase.sh`**, not committed. Regenerate before
  each cutover dry-run so they reflect the latest source state.

The split exists because policies reference functions and tables that must
already exist when CREATE POLICY runs; importing in three passes avoids
ordering crashes that a single `pg_dump` restore produces.
